## 备注说明

### 1. Python 环境管理

- WorkBuddy 使用托管的 Python 和 Node.js 环境（managed），位于 `C:\Users\Administrator\.workbuddy\binaries\`
- 建议优先使用 managed 版本，以确保环境隔离和一致性
- 当前 pip 使用的是系统 Python 3.11.9，需要注意包安装路径

### 2. 包安装建议

对于项目开发，建议：

```bash
# 为项目创建独立的虚拟环境
python -m venv venv

# 激活虚拟环境
# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

# 安装项目依赖
pip install -r AutoSmoke/data_access/config/requirements.txt
```

### 3. 更新 pip

当前 pip 版本为 24.0，有新版本 26.1.2 可用。建议更新：

```bash
python -m pip install --upgrade pip
```

### 4. 测试框架说明

- **Airtest**: 主要用于游戏自动化测试，支持图像识别、UI 自动化
- **POCO**: UI 自动化框架，可以与 Airtest 配合使用
- **pytest**: 单元测试框架，用于编写和执行测试用例

### 5. 移动设备测试支持

- **iOS**: 已安装 `facebook-wda` 和 `tidevice`，支持 iOS 设备自动化测试
- **Android**: 需要安装 Android SDK (ADB) 才能进行测试

## 维护建议

1. **定期更新依赖包**：
   ```bash
   pip list --outdated
   pip install --upgrade [package_name]
   ```

2. **使用虚拟环境**：避免全局安装包，为每个项目创建独立环境

3. **固定版本号**：在生产环境中，建议固定依赖包的版本号以避免兼容性问题

4. **文档同步更新**：当添加或删除依赖时，及时更新本文档
