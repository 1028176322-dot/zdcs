> 最后更新：2026-06-15（v4.0）

---

## 一、项目概述

**项目名称：** AutoSmoke — Unity Android 游戏自动化测试系统

**核心目标：** 构建通用、智能的Unity游戏自动化测试系统，实现自动获取游戏状态、自动探索UI、自动执行测试、自动发现异常。

**技术栈：**
- Unity 2022.3.62f3 + Poco SDK
- Python 3.11+ (Pillow, NumPy, Pandas, Flask)
- Windows API (win32gui/win32ui)
- PyQt6 / Flask Web IDE
- LLM Integration (GPT/Claude/Qwen)
- **新增：** C# Editor 脚本（GameView定位 + 点击注入）

**设计原则：**
- 通用性优先：所有功能必须支持不同的Unity游戏项目
- 模块化设计：各功能模块独立，可单独启用/禁用
- 可扩展性：支持插件机制，方便添加新功能
- **新增：** 不修改游戏业务代码，允许低侵入Unity Editor脚本

### 7.3 当前问题与解决方案

#### 问题1：ClickContent文本字段为空

- **现象**：dump结果显示34个可点击节点，但所有`text`字段都为空
- **原因**：ClickContent节点本身没有`Text`组件，文本存储在兄弟节点或祖先节点的子节点中
- **解决方案**：✅ 已实现Python侧修复算法（`find_nearby_text()`）
- **当前状态**：✅ 已修复，待实际游戏验证

#### 问题2：UnityNode.cs修改未生效

- **现象**：修改`UnityNode.cs`后，Unity Console未输出调试信息
- **原因**：Unity未重新编译C#脚本，或Poco RPC服务器未重启
- **解决方案**：优先使用Python侧修复（方案B），不依赖Unity修改
- **当前状态**：✅ 已采用方案B

#### 问题3：坐标转换精度

- **现象**：Poco返回的坐标是归一化坐标（0-1），需要转换为屏幕像素坐标
- **解决方案**：使用公式 `screen_x = normalized_x * game_window_width`
- **当前状态**：✅ 已实现，待进一步验证精度

#### 问题4：副屏点击偏移

- **现象**：副屏下点击坐标偏移，点到错误位置
- **原因**：`GetSystemMetrics(0/1)`只返回主屏尺寸，不包含副屏
- **解决方案**：✅ 已加入虚拟屏幕偏移 `screen_offset`，使用`GetSystemMetrics(76/77/78/79)`
- **当前状态**：✅ 已修复

#### 问题5：OCR功能未启用

- **现象**：`game_content_vision.py`中OCR功能因缺少依赖未启用
- **原因**：未安装`pytesseract`和`Tesseract-OCR`
- **解决方案**：需手动安装
- **当前状态**：⏳ 待安装
